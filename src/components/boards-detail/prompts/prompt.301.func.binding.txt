1) 테스트 제외 라이브러리
- jest

2) 테스트 조건

- 페이지가 완전히 로드된 후 테스트할 것.

  - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

3) 테스트 API 조건
3-1) 데이터
- 실제 API 데이터를 사용할 것.

3-3) 실패 시나리오

4) 데이터 조건
    - 저장소: 백엔드 서버(GraphQL API)
    - 요청방식: fetchBoards qurey
    - 요청 파라미터 구조 :

        query fetchBoard($boardId:ID!) {
          fetchBoard(boardID: $boardId) {
            _id: ID!
writer: String
title: String!
contents: String!
youtubeUrl: String
likeCount: Int!
dislikeCount: Int!
images: [String!]
boardAddress: BoardAddress
user: User

          }
        }

핵심요구사항) 다이나믹 라우팅된 게시글 상세페이지의 [boardId]를 추출하여 Mock 데이터를 제거하고, **Apollo Client를 통해 fetchBoard Query API 데이터**를 바인딩할 것.

1) 바인딩할 데이터
- Apollo Client useQuery(fetchBoard, { variables: { boardId } }) 결과
- 반환된 fetchBoard 객체를 사용

2) 상세내용
- 제목: fetchBoard.title
- 작성자: fetchBoard.writer
- 작성일: fetchBoard.createdAt
- 이미지: fetchBoard.images
- 내용: fetchBoard.contents
- 유튜브링크: fetchBoard.youtubeUrl

3) CSS 처리
- 제목, 내용, 이미지 등 화면에 맞게 overflow 처리 및 스타일 적용

4) 테스트
- 로컬스토리지 모킹 없이 실제 API 데이터를 테스트
- Playwright 테스트에서 boardId 파라미터 기반 API 호출 후 렌더링된 데이터를 검증
- data-testid를 활용하여 각 필드 검증
